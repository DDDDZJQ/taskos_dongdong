# Workflow: Retrospect（手动复盘）

从周快照数据实时生成趋势分析，帮助用户回顾过往状态。**纯只读操作**，不走 journal，不修改任何文件。

---

## 一、触发词

- 回顾 / 看看趋势 / 看看历史 / 我最近怎么样 / 复盘一下最近

> **与 Weekly Review 的区分**：
> - "周复盘 / 本周怎么样" → Weekly Review（更新 progress、生成当周快照）
> - "回顾 / 看看趋势 / 复盘一下最近 N 周" → Retrospect（只读分析，不更新数据）
> - 区分关键词：用户意图是"更新本周状态" → review；意图是"看历史趋势" → retrospect

---

## 二、输入

用户可以指定范围，否则默认看所有可用数据：
- "回顾最近 4 周"
- "看看上个月的趋势"
- "我最近怎么样"（默认：所有可用周快照）

---

## 三、执行流程

### 步骤 1：读取周快照文件

根据用户指定范围（或全部）读取 `reviews/YYYY-Www.md`（含 `reviews/archive/` 中的旧文件）。

**兼容性**：如果遇到旧格式文件（frontmatter `type: review`），也能提取其中的项目进展和当周快照数据。

### 步骤 2：读取 done 文件

对应时间范围的 `done-YYYY-MM.md`，用于补充完成明细。

### 步骤 3：读取决策记录

从 `.journal.md` 和 `.journal-YYYY-MM.md` 中提取 `[decision]` 标记的条目。

### 步骤 4：实时生成分析报告

---

## 四、报告内容（AI 实时生成，不预存）

### 4.1 趋势速览

```
趋势速览（近 N 周）：
- 完成量：12 → 8 → 15 → 10 → 13（本周）↑
- carry 积压：3 → 4 → 2 → 5 → 3 ↓
- core 推进：精读 42%→50%→55% ↑ / 播客 18%→30% ↑ / 雅思 20%→20% →（停滞）
- 精力记录：充沛 → 正常 → 低迷 → 正常 → 充沛
- 上下文分布变化：@阅读 占比从 25% 提升到 40%
```

**数据来源**：周快照的 `completed` / `total_carry_tasks` / 项目状态 progress / `energy` / 上下文分布。

### 4.2 项目进展时间线

从快照中提取每周 progress 变化，按项目列出时间线。

```
精读《动力取向心理治疗》: 42% → 45% → 50% → 55% → 58%（+16%，8周）
播客第5期: 18% → 25% → 30% → 35%（+17%，8周）
雅思7.5: 20% → 20% → 20% → 20%（停滞 4 周 ⚠️）
```

对于停滞的 core 项目，温和提出：
> "雅思连续 N 周没推进，是暂时搁置还是需要调整策略？"

### 4.3 关键决策回顾

从 journal `[decision]` 中提取时间范围内的决策：

```
关键决策：
- 2026-04-15: drop project「雅思7.5」reason: 优先级调整
- 2026-04-20: 升级「播客第5期」为 core
- 2026-05-01: 调整「精读《...》」deadline 延后 2 周
```

### 4.4 精力-产出关联

```
精力-产出关联：
| 精力 | 出现次数 | 平均完成量 | 平均 est |
|------|---------|-----------|---------|
| 充沛 | 3 周 | 15 条 | 12h |
| 正常 | 4 周 | 10 条 | 9h |
| 低迷 | 1 周 | 8 条 | 6h |

结论：波动正常，不必焦虑。低迷周完成量低是正常的。
```

### 4.5 月度对比（如跨月）

```
月度对比：
- 本月平均完成量：11.2 条/周
- 上月平均完成量：9.5 条/周
- 上上月：12.0 条/周
```

### 4.6 AI 观察与建议

基于数据的温和建议，遵循"诤友"人格：
- 先肯定做得好的（哪些 core 项目在推进）
- 再指出需要注意的（停滞项目、持续高 carry）
- 给出可操作的建议

---

## 五、关键原则

1. **纯只读**：不写任何文件，不走 journal，不更新 INDEX
2. **实时生成**：每次复盘都从最新数据计算，不依赖预存报告
3. **数据驱动**：所有结论基于周快照中的结构化数据
4. **不自动触发**：只在用户手动请求时执行
5. **语气温和**：先正向再指出问题，像朋友聊天

---

## 六、边界情况

### 快照数据不足
- 只有 1-2 周数据 → 跳过趋势分析，只呈现当前状态
- 某些周缺失快照（用户长期未用） → 标注"数据缺失"，不编造

### 没有决策记录
- journal 中无 [decision] 条目 → 跳过"关键决策回顾"段

### 旧格式兼容
- 遇到 `type: review` 的旧文件 → 尽可能提取"项目进展"和"当周快照"段的数据

## 八、Strategy 检视提醒（v1.2 新增）

复盘报告输出完毕后：
- 检查是否有 active strategy project 的 last_reviewed 超过 3 个月
- 如有 → 追加提醒："顺便说一下，「xxx」路线图已经 N 个月没检视了，要看看进度吗？"

- 用户说否 → 不做任何操作

## 九、价值对齐审计（Value Alignment Audit）（v1.5.0 新增）

### 理论基础

ACT 价值罗盘：区分"目标"和"价值"——目标是可完成的终点，价值是持续的方向。忙着完成目标但走的方向不是自己真正在乎的，是常见的"效率陷阱"。

### 触发条件

- 每季度（约 12 周）自动触发一次
- 或用户说"看看价值对齐"/"我的时间花在哪了"/"值不值"时手动触发
- AI 通过比对 `INDEX.last_value_audit` 与今天日期判断（差值 ≥ 84 天则触发）
- 首次触发：`last_value_audit == null` 且快照数据 ≥ 8 周时
- **嵌入复盘报告末尾**或作为独立会话，不在启动时触发

### 数据采集

从近 12 周（或所有可用周）的 done-*.md 和快照中统计：
- 各 area 完成任务数占比
- 各 area 投入 est 时间占比
- yearly_intent 非空的 area 的实际推进情况

### 呈现格式

```
📊 过去 3 个月的时间投入分布：

按领域（area）：
  心理学     ████████████░░░ 45%  (yearly_intent: "考取初级证书")
  自媒体     ████████░░░░░░░ 30%  (yearly_intent: "稳定周更")
  英语学习   ███░░░░░░░░░░░░ 12%  (yearly_intent: "雅思7.5")
  _其他      ██░░░░░░░░░░░░░ 13%

你在心理学上的声明意图是"考取初级证书"——实际投入也最多，方向一致。
英语学习声明了"雅思7.5"但只占 12% 的投入——这符合你的预期吗？
```

### AI 行为原则

- **只呈现事实，不做价值判断**——"英语只占 12%"不意味着"你做得不好"
- **开放式提问**——"这符合你的预期吗？"而非"你是不是应该多花时间"
- **尊重选择**——用户说"我就是不想花太多时间"→ 完全接受，不追问
- **允许修正意图**——如果用户说"其实我已经不想考了"→ 引导更新 area 的 yearly_intent
- **完成后更新 INDEX.last_value_audit 为今天日期**

### 冷却规则

- 最多每 12 周触发一次
- 用户上次跳过/不想做 → 冷却延长到 16 周
- 不在启动时触发

### 情绪模式觉察（附带功能）

如果快照中有 ≥ 4 周的 mood 数据，价值审计时也展示情绪-产出关联：

```
情绪-产出关联（近 12 周）：
| 情绪 | 出现次数 | 平均完成量 | 
|------|---------|-----------|
| 平稳 | 5 周 | 11 条 |
| 有动力 | 3 周 | 14 条 |
| 焦虑 | 2 周 | 12 条 |
| 低落 | 1 周 | 6 条 |
```

注意：不做因果归因（"焦虑导致低产出"），只呈现关联。
