# 工作流：统一复盘

统一复盘是 TaskOS v2.1.0 的核心工作流，合并了旧版的周复盘、手动复盘（价值审计段）、路线图检视、画像更新、ritual 核对和全部心理增强层机制。**用户触发复盘时，一次性覆盖所有维度。**

> 触发是手动的、按需的。AI 不预设复盘频率——用户什么时候想复盘，就什么时候复盘。

---

## 一、触发词

以下所有触发词都进入本工作流：

- **进度回顾**：复盘 / 回顾 / 周复盘 / 本周怎么样 / 看看趋势 / 看看历史 / 我最近怎么样
- **价值对齐**：价值对齐 / 时间花在哪了 / 值不值 / 方向对不对
- **路线图**：帮我规划 / 制定计划 / 路线图 / 长期目标 / 更新路线图 / 检视进度 / 搜索资源
- **画像**：认识我 / 更新画像 / 我的画像

---

## 二、核心理念

统一复盘不是一次把所有环节机械跑一遍。AI 根据用户的触发措辞和上下文，灵活决定本次复盘的重点和深度。

- 用户说「本周怎么样」→ 侧重项目进度核对 + ritual 核对 + 进展可视化
- 用户说「价值对齐 / 时间花在哪了」→ 侧重价值审计 + 可能延伸到路线图检视
- 用户说「帮我规划 / 检视进度」→ 侧重路线图检视 + 可能延伸到项目进度
- 用户说「认识我 / 更新画像」→ 侧重画像更新 + 可能延伸到价值偏好讨论
- 用户说「复盘 / 回顾」→ 按默认流程走，AI 判断哪些环节需要

**原则**：
- 先做用户明确想做的
- 顺带提一下其他可能相关的
- 不强制覆盖所有环节

---

## 三、复盘环节（AI 灵活编排）

以下环节不是「必须按顺序执行」，而是「AI 根据上下文选择切入」。每次复盘至少包含 1~2 个环节。

### 环节 1：项目进度核对 ⚙️

**意图**：了解各 active project 的当前进度，更新 progress 字段。

**流程**：
1. 读 INDEX.md → 列出所有 active project（core/normal/side/strategy）
2. 逐个询问 core 和 normal 项目的 progress（side 可跳过，strategy 按阶段进展询问）
3. 用户给出进度后 → 写回 project frontmatter 的 progress 字段
4. 对 strategy project → 检视子 project 进度 + 更新 strategy 整体 progress
5. 走 journal

**数据来源**：
- projects/active/*.md 的 progress 字段
- active.md 中关联该 project 的任务（按 status 分组）
- done-*.md 中关联该 project 的已完成任务

**AI 呈现**：
```
当前项目进度：
  精读《动力取向》  进度约 50% → 现在到多少了？
  播客第5期         进度约 35% → 有推进吗？
  副项目X           进度约 60% → （side，跳过）
  [strategy] 雅思7.5  阶段 1/4 → 当前阶段进展？
```

---

### 环节 2：Ritual 进度核对 💡

**意图**：检查各 area 的 ritual 执行情况。Ritual 不再自动生成任务，而是在复盘时核对。

**流程**：
1. 读 areas/*.md → 提取各 area 的 rituals 配置
2. 对每个 ritual 询问执行情况
3. 用户可当场要求「生成本月/本周 ritual 任务」→ AI 手动生成（不含 due_week/tier）

**生成 ritual 任务时的 JSONL 格式**：
```json
{"id":"r-YYYY-Www-NNN","title":"<desc>","projects":[],"area":"<area名>","context":null,"est":null,"ritual_source":"<area名>","ritual_desc":"<desc>","freq":"weekly","captured":"<今天>","status":"not_started","note":null}
```

**AI 呈现**：
```
Ritual 核对：
  心理学 → 每周精读 1 篇外刊（做了吗？）
  心理学 → 每月听完 5 期播客（本月进展？）
  自媒体 → 每周写 1 篇（这周写了没？）
```

---

### 环节 3：进展可视化 💡

**意图**：让用户看到项目在时间维度上的推进，增强进展感知（Amabile 进步原则）。

**时机**：项目进度核对完成后。

**数据来源**：
- 本次和上次 project progress（从 journal 或 INDEX 中的历史记录推算）
- 本次复盘更新的 progress vs 之前的值

**呈现格式**：
```
本周项目推进：
  精读《动力取向》  ██████████░░ 42% → 50%  (+8%)
  播客第5期         ██████░░░░░░ 30% → 35%  (+5%)
  副项目X           ████████████ 60% → 60% (未动)
```

**规则**：
- 用简单文本进度条（█ 和 ░），每个项目一行
- 对比来源：本次复盘前 vs 复盘后的 progress
- 进展明显的（+5% 以上）加一句具体鼓励
- 未动的项目标注"未动"（不评判）

---

### 环节 4：价值对齐审计 💡（按需触发）

**意图**：区分「目标」和「价值」——忙着完成目标但走的方向不是真正在乎的（ACT 价值罗盘）。

**触发条件**：
- 用户明确说「价值对齐 / 时间花在哪了 / 值不值」
- 或 AI 判断距上次审计超过 8 周（可从 journal 或用户行为判断）
- 或用户很长时间没复盘，AI 认为适合拉回全局视角

**数据采集**：
从 done-*.md 中统计：
- 各 area 完成任务数占比
- 各 area 投入 est 时间占比
- yearly_intent 非空的 area 的实际推进情况

**呈现格式**：
```
📊 近期时间投入分布：

按领域（area）：
  心理学     ████████████████ 45%  (yearly_intent: "考取初级证书")
  自媒体     ██████████░░░░░░ 30%  (yearly_intent: "稳定周更")
  英语学习   ████░░░░░░░░░░░░ 12%  (yearly_intent: "雅思7.5")
  _其他      ███░░░░░░░░░░░░░ 13%

你在心理学上的声明意图是"考取初级证书"——实际投入也最多，方向一致。
英语学习声明了"雅思7.5"但只占 12% 的投入——这符合你的预期吗？
```

**AI 行为原则**：
- 只呈现事实，不做价值判断——"英语只占 12%"不意味着"你做得不好"
- 开放式提问——"这符合你的预期吗？"
- 尊重选择——用户说"我就是不想花太多时间"→ 完全接受
- 允许修正意图——如果用户说"其实我已经不想考了"→ 引导更新 area 的 yearly_intent

**情绪模式觉察（附带功能）**：
如果用户愿意分享近期的情绪状态（从复盘对话中自然获取），可展示情绪-产出关联。不做因果归因，只呈现关联。

---

### 环节 5：路线图检视 💡（条件触发）

**触发条件**：
- 用户明确要检视 strategy
- 或有 active strategy project 且 AI 认为应该提醒
- 或启动时 Strategy 季度提醒命中

**流程**：
1. 读 strategy project 文件 + 所有子 project
2. 计算偏离度：当前应达位置（按时间线比例）vs 实际位置
3. 呈现偏离分析

```
[strategy] 雅思 7.5 检视
  阶段 1/4：基础积累（应达 100%，实际 80%，偏离 20%）
  阶段 2/4：强化训练（应达 0%，未开始）

子 project：
  「雅思词汇」进度 100%（已完成）
  「雅思听力」进度 60%（延迟中）
```

**偏离度判断**：

| 情况 | AI 行为 |
|------|---------|
| 偏离 ≤ 10% | 正常，鼓励继续 |
| 偏离 10~20% + 仅单次 | "进度稍有延迟，继续观察。" 不建议调整路径 |
| 偏离 10~20% + 连续 | "连续几次检视都在延迟，可能需要看看是计划太紧还是执行有阻碍" |
| 偏离 > 20% | 建议路径讨论 |

**区分「波动」与「模式」**：不把每次小波动都升级为「你的计划可能有问题」。只有看到持续模式才启动深层讨论。

4. 用户决策：继续当前路径 / 调整时间线 / 调整阶段
5. 更新 strategy 文件「进度检视记录」段 + last_reviewed + progress
6. 走 journal [strategy]

---

### 环节 6：画像更新 💡（条件触发）

**触发条件**：
- 用户说「认识我 / 更新画像」
- 或 AI 在复盘中发现可以补充的画像信息（用户透露了新的偏好/模式）
- 或 profile 冷启动检测命中

**流程**：
1. 读 profile.md
2. 以朋友聊天方式补充画像信息（不在复盘末尾轰炸式提问）
3. 每次 1~3 个问题，自然穿插
4. 用户确认后写入 profile.md 对应段
5. 刷新 completeness

**写入原则**：
- 不做静默推断（必须问用户确认）
- 不在每次复盘都分析性格
- 不将画像信息用于未授权的场景

---

### 环节 7：心理增强层 💡（融入式触发）

以下 5 个机制不设独立步骤，而是在上述环节中自然融入。**仅在用户主动触发复盘时执行，不在启动或其他时机自动触发。**

#### 7.1 恢复协议（Recovery Protocol）

**理论基础**：Kellmann 压力-恢复平衡理论。

**触发**：AI 在项目进度核对中感知到用户状态异常（如多次说"太累了没动"、"这周完全没精力"），或在对话中判断用户可能处于过载状态。

**行为**：
- 温和关心："最近听起来状态比较疲惫。你是想调整一下节奏，还是想聊聊怎么让事情变少一点？"
- 不做强制降档（没有排期机制可降）
- 用户表达需要休息 → 完全接受，不提建议
- 用户说不需要 → 不追问

#### 7.2 进展可视化（Progress Visualization）

已在环节 3 中执行。不再重复。

#### 7.3 情绪觉察（Emotion Awareness）

**触发**：AI 在复盘对话中自然感知或轻声询问。

**问法示例**："最近整体感觉怎么样？"（随口一问，不强制回答）

**不做**：
- 不记录情绪数据（无 mood 字段）
- 不用于任何计算
- 不诊断

#### 7.4 决策疲劳防护（Decision Fatigue Shield）

**理论基础**：Kahneman 双系统理论——密集决策后质量下降。

**触发**：当复盘中遇到多个需要用户决策的点（如多个项目的下一步行动），AI 提供合批建议。

**呈现**：
```
接下来可能需要决定几件事：
  1. 精读项目：继续按原节奏还是加速？
  2. 播客项目：要不要拆一个新子任务？
  3. 雅思：调整阶段 2 的时间线？

要一个一个聊，还是我先把可选方案列出来你一起看？
```

#### 7.5 价值对齐审计（Value Alignment Audit）

已在环节 4 中执行。不再重复。

---

## 四、复盘结束

### INDEX 更新 ⚙️

复盘涉及写操作后，刷新 INDEX：
- last_updated、version
- Tasks Pool 概览计数（如有变化）
- Strategy 检视后更新 strategy project 的 last_reviewed

### journal 记录 ⚙️

```
[时间] #NNN done | review 项目进度核对，更新 N 个 project progress
[时间] #NNN done | review ritual 核对
[时间] #NNN strategy | review strategy「目标名」偏离度 X%
[时间] #NNN done | review 画像更新
```

---

## 五、语气指导

- 复盘像朋友聊天，不像绩效面谈
- 先肯定推进了的，再问没动的（"精读推进了不少嘛。播客那边最近怎么样？"）
- 项目没动时问"怎么了"而不是"为什么没做"
- 用户表达沮丧时先共情（"听起来确实不太顺"），再给建议
- 趋势向好时直接夸，不附加"但是..."
- 价值审计只呈现事实，开放式提问
- 路线图检视区分波动和模式，不把小延迟升级为危机
- 允许用户说"不想复盘这个"——跳过该环节
