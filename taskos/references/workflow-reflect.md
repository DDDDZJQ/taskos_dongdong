# Workflow: Adhoc Reflection（随手反思）

让用户在任何时候记录想法、感受、洞察，不必等到周复盘。记录到 `reflections.md` 滚动文件，周复盘时自动汇总。

---

## 一、触发词

- 记个反思 / 随手反思 / 想到一件事 / 记一下感想 / 突然想到

> **与 Weekly Review 微反思的区分**：
> - 本工作流：用户**主动随手**记录，任何时候都能用，不问问题
> - 周复盘微反思：在周复盘流程末尾由 AI 提问，属于 workflow-weekly.md

---

## 二、与任务捕获（capture）的区分（关键）

`记一下` 这个词有歧义，必须先判断用户意图：

| 内容性质 | 判断特征 | 去向 |
|---------|---------|------|
| 待办事项 | 动词前置、有"要做"性质（"记一下：联系督导"） | inbox（workflow-capture.md） |
| 想法/感受/认识 | 陈述一个观察、感受、觉察（"早上效率更高"） | reflections.md（本工作流） |

**判断规则**：
- 单独的"记一下 XX"默认走 **capture**（保持向后兼容，不破坏老习惯）
- 明确的"记个反思 / 记一下感想 / 随手反思 / 突然想到"走**本工作流**
- 内容模糊、两可时，反问一句："这是想记一件待办，还是记一个想法/感受？"

---

## 三、执行流程

### 记录规则（⚙️ 规则层，数据一致性）

1. 读 `reflections.md`（不存在则创建，含标题 `# Reflections` + 空 JSONL 区块）
2. 生成 id：`refl-YYYYMMDD-NNN`，扫描 reflections.md 当天已有行确保 NNN 唯一（从 001 起）
3. 构造 JSONL 行，把用户原话**一字不改**写入 `content`：
   ```json
   {"id":"refl-20260529-001","date":"2026-05-29","week":"2026-W22","content":"<用户原话>"}
   ```
   - `date` = 今天（YYYY-MM-DD）
   - `week` = 今天的 ISO 周（用日期计算，不能字符串递增）
4. append 到 JSONL 区块结束行的前一行
5. 走 journal（单行）：`[时间] #NNN done | reflect refl-20260529-001 → reflections.md`

> **注意**：reflections.md 不计入 INDEX 的任何计数，**不刷新 INDEX**（它不属于任务池/项目体系，是独立的反思流）。仅走 journal 即可。

### 交互风格（💡 指导层）

6. 回一句轻量确认："记下了。"（或自然的等价表达）
7. **不追问、不展开、不分析、不点评**——保持低摩擦捕获

---

## 四、AI 行为红线

- ❌ 不对反思内容做即时点评、分析、追问原因
- ❌ 不引导用户"深挖一下"——破坏低摩擦捕获
- ❌ 不主动提醒用户"该反思了"——反思是用户主动行为，不催不推
- ✅ 仅在内容明显是待办事项时，提示一句："这看起来像个待办，要不要记成任务？"

---

## 五、reflections.md 数据格式

```markdown
# Reflections
## ~~~ JSONL 区块开始 ~~~
{"id":"refl-20260529-001","date":"2026-05-29","week":"2026-W22","content":"早上写东西效率明显比晚上高"}
{"id":"refl-20260530-001","date":"2026-05-30","week":"2026-W22","content":"和督导聊完后对个案概念化清楚多了"}
## ~~~ JSONL 区块结束 ~~~
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | `refl-YYYYMMDD-NNN` |
| `date` | string | ✅ | 记录日期（YYYY-MM-DD） |
| `week` | string | ✅ | 记录时所属 ISO 周 |
| `content` | string | ✅ | 用户原话，不二次加工 |

JSONL 操作规则同 tasks/active.md（区块边界固定、每行合法 JSON 占满一行、坏行跳过 + journal [align]）。

---

## 六、与周复盘的闭环

周复盘的微反思环节会自动读取本周的散记（`week == current_week`），先回顾给用户看、汇总进当周快照的 `## 本周反思` 段（用 `[随手]` 前缀标注），再问当周的微反思问题。详见 workflow-weekly.md 周复盘 §微反思。

**散记汇总后保留在 reflections.md 不删除**，作为长期反思流。

---

## 七、边界情况

- reflections.md 不存在 → 首次记录时自动创建
- 用户一次说了多条想法 → 可拆成多行分别记录，或合并为一行（AI 判断，倾向保留用户原始表述结构）
- 内容明显是待办 → 按"与 capture 区分"规则处理，不硬塞进 reflections.md
