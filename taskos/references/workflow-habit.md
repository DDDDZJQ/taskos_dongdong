# 工作流：习惯打卡系统

> v1.7.0 新增。分层（核心层/观察层）+ 批量打卡（默认全过只报例外）+ 旋转门毕业。
> 独立于许愿卡 Streak；懒加载，启动不读 habits.md。

---

## 一、触发词

- 打卡 / 习惯打卡 / 今天打卡 → 批量打卡
- 加个习惯 / 新习惯 / 想养成 xxx → 创建习惯
- 我的习惯 / 习惯进展 / 连续多少天了 → 查询（只读）
- 习惯毕业 / 让 xxx 毕业 → 毕业流程
- 把 xxx 降到观察层 / 升到核心层 → 层间流转

> 与许愿卡 Streak 触发词消歧：Streak 用"连续纪录 / 接受挑战 / 本周挑战"等挑战语境词；习惯系统用"打卡 / 习惯"。若用户单说"打卡"且两系统都有可打项，默认走习惯系统（日常高频），可一句话确认。

---

## 二、两层结构

```
habits.md
├── 核心层（core）  —— 硬上限 3 个
│   · 每日批量打卡，断签温和提示
│   · 计连续天数（streak）+ 总次数（total）
│   · 自动化后「毕业」→ 腾位
│
└── 观察层（watch） —— 不限量
    · 做了顺口记一次，不催不提醒
    · 只累计 total，streak/best_streak 恒 null，永不断签
    · 可手动升级进核心层（若核心层有空位）
```

| 层 | 数量 | 计数 | 断签 | 打卡方式 |
|----|------|------|------|----------|
| 核心层 core | 硬上限 3 个 | streak + total | 断签温和提示（不归零羞辱） | 进每日批量打卡 |
| 观察层 watch | 不限量 | 只累计 total | 永不断签 | 顺口提及单记，不进批量 |

## 三、⚙️ 与现有机制的边界（按目的划界，防混乱）

| 机制 | 目的 | 心态 | 典型例子 | 终点 | 数据位置 |
|------|------|------|----------|------|----------|
| **习惯系统**（本工作流） | 养成自动化——目标是有天不用再记 | 日常、低压、做了就记 | 喝水、拉伸、冥想 | 毕业移除 | habits.md |
| 许愿卡 Streak（v1.4.0） | 游戏化激励——刻意维持连续、攒卡兑奖 | 挑战、成就、看数字涨 | "连续 100 天健身挑战" | 想停就停 | rewards/streaks.md |
| area ritual（v1.2） | 周期性事项纳入周计划排期 | 任务、进任务池 | 每周精读 1 篇外刊 | 随 area 存续 | active.md（r- 行） |

**划界判断**（用户新增习惯/打卡语义模糊时，AI 反问一句）：
- 想内化成不用想的日常 → 习惯系统
- 想挑战连续天数、攒卡 → 许愿卡 Streak
- 想纳入每周排期、当任务推进 → area ritual

## 四、创建习惯

> 习惯**不走 Gatekeeper**（Gatekeeper 是 project 门禁）。习惯有自己的轻量准入：核心层 3 闸为主约束，观察层几乎零门槛。

1. 用户说"加个习惯：冥想"
2. **判断层级**：默认进**观察层**（低压起步）；用户明确说"要每天盯/进核心"才进核心层
3. 若要进核心层且核心层已满 3 个 → 提示"核心层满了（3 个），要先让某个毕业/降级，还是先放观察层？"
4. **边界判断**（§三）：若像"想挑战连续天数" → 提示考虑用许愿卡 Streak；若像"想纳入周排期" → 提示用 ritual
5. 读 habits.md（不存在则建：标题 + 空 JSONL 区块），生成 ID `h-YYYYMMDD-NNN`（扫当天已有，NNN 从 001 递增）
6. append 新行：`status:"active"`，`total:0`，`last_checkin:null`；核心层 `streak:0,best_streak:0`，观察层 `streak:null,best_streak:null`
7. 刷新 INDEX `## Habits 概览`；journal `[habit-create]`

## 五、批量打卡（核心机制，💡 指导层为主）

**核心规则："默认全过，只报例外"** —— 只作用**核心层**。

| 用户说 | AI 行为 |
|--------|---------|
| "打卡" / "习惯打卡" | 核心层**所有** active 习惯今日标记完成 |
| "打卡，冥想没做" | 除冥想外核心层全部完成；冥想今日不打 |
| "打卡，只做了运动" / "今天只打卡运动" | 仅运动完成，其余不打 |
| "今天拉伸了"（拉伸在观察层） | 观察层"拉伸"累计 +1（顺口记，无需"打卡"指令） |

> **边界：批量打卡指令只作用核心层，不含观察层。** 观察层始终靠用户顺口提及单独累计——这是观察层"做了就记、不催不盯"定位的必然结果。想把观察层习惯纳入每日批量，应先升级到核心层。

### ⚙️ 规则层（数据一致性）

1. 批量打卡是多条写操作，走**单次 journal 事务**（`[in_progress]` → 逐条更新 → 同 op 编号 `[done]`），标记 `[habit-checkin]`
2. **单习惯打卡字段更新**（核心层与观察层共用）：
   - `total += 1`、`last_checkin → 今天`
   - 核心层额外：先断签检测（§六）——未中断 `streak += 1`，中断则归 0 后按本次置 1；本次打卡后 streak 至少为 1；如 `streak > best_streak` 则 `best_streak = streak`
   - 观察层：`streak / best_streak` 恒 null，不动
3. 每个习惯当天只记一次（重复"打卡"指令幂等，不重复 +1）
4. 当天已打卡的习惯再次"打卡" → 跳过并轻量告知（"今天的已经记过啦"）
5. 刷新 INDEX `## Habits 概览`

### 💡 指导层（交互）

6. 打卡后一句轻量汇报："今天核心 3 个都打上了 ✓"或"打上了运动、阅读，冥想留着明天"
7. 不展开、不追问"为什么冥想没做"（除非用户主动说）

---

## 六、⚙️ 断签检测（仅核心层）

- 打卡时检查：`今天 − last_checkin > 1 天` → 连续中断
- 中断处理：`streak → 0`（温和告知，不羞辱："连续记录断在了 N 天，重新开始也很好"），随即按本次打卡置 `streak = 1`
- 观察层不做断签检测（streak 恒 null）

---

## 七、毕业（旋转门，💡 指导层）

**目的**：让核心层 3 个位置流动起来，固化的习惯腾位给新的。

- **毕业锚点**：连续达标 ~66 天（Lally 研究均值）作**参考锚点，非硬阈值**——最终以用户主观"觉得不用刻意了"为准
- **AI 触发**：核心习惯连续天数接近/超过 66 天时，温和询问："『喝水』你已经连续 70 天了，感觉是不是不用刻意提醒也会做了？要不要让它毕业，腾个位置给新习惯？"
- **毕业动作**：该习惯 `status → graduated`，移出核心层活跃打卡列表（记录保留）；核心层空出 1 位
- **奖励**：毕业时发 1 张许愿卡作里程碑庆祝（走许愿卡 earn，source: "habit-graduate"；rewards/ 不存在则跳过发卡，仅文字祝贺，不阻塞）
- **腾位提示**（不强制）：毕业后可顺势一句"核心层空出 1 个位置，有没有想新养成的？也可以先空着"——不催不强塞
- **用户拒绝毕业**：保留在核心层继续打卡，不强制；冷却一段再问
- journal `[habit-graduate]`

> 毕业是"恭喜"不是"踢出"。措辞传递成就感。

---

## 八、层间流转

- **观察层 → 核心层**（升级）：用户主动；若核心层已满 3 个 → 提示"核心层满了，要先让某个毕业或降级吗？"；升级后补全 streak/best_streak（从 null 改 0 起算）
- **核心层 → 观察层**（降级）：用户主动（不想每天盯了但还想记着）；streak/best_streak 改 null
- 流转走整行替换 + journal（`[habit-create]` 同类，注明 layer 变更）+ 刷新 Habits 概览

## 十、⚙️ 奖励联动（唯一接触点）

许愿卡 v1.4.0 定位"非胡萝卜"（避免外部奖励侵蚀内在动机）。习惯系统据此：
- ❌ **日常打卡不发卡**——每天打卡发卡会把"想内化"变成"为卡而做"
- ✅ **仅核心习惯毕业时发 1 张**——养成成功的里程碑庆祝
- 发卡走许愿卡 ledger（source: "habit-graduate"），复用 earn 流程；rewards/ 不存在则跳过

## 十一、即时核查（习惯写操作后）

| 操作 | 检查 |
|------|------|
| 创建习惯 | id 当天唯一（h-）；layer/status 合法；核心层未超 3（超出告警不强拦）；观察层 streak/best_streak 为 null；journal [habit-create]；Habits 概览刷新 |
| 批量打卡 | 单次事务；每习惯当天只 +1；核心层 streak 递增/断签正确；total/last_checkin 已更新；journal [habit-checkin]；Habits 概览刷新 |
| 毕业 | status→graduated；核心层计数 -1；rewards 存在则发 1 张（source habit-graduate）；journal [habit-graduate] |
| 层间流转 | layer 已改；升核心层时核心层未超 3；streak/best_streak 按层调整（核心 0 起 / 观察 null）；journal；Habits 概览刷新 |

---

## 十二、不做的事

- ❌ 观察层不设断签/连续（给随意做的事加压，违背初衷）
- ❌ 日常打卡不发卡（守"非胡萝卜"哲学）
- ❌ 习惯不进启动校验（懒加载）
- ❌ 习惯不走 Gatekeeper（低门槛；核心层 3 闸已够）
- ❌ 不做习惯提醒/推送（"做了就记"是用户主动）
- ❌ 不强制习惯挂 project/area
